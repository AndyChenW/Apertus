Apertus Instructions

Apertus can support most coins with the proper settings; we have pre-loaded the settings for Bitcoin, Litecoin, Dogecoin, Mazacoin, and Anoncoin.


I. Synchronize your Wallet(s)

Apertus requires the full wallet/blockchain of the coin you are archiving on.

~Download the full wallet (and thus the blockchain) of the coin. (ex. go to Bitcoin.org, click “Get Started with Bitcoin”, then click “Choose your wallet”, download & run “Bitcoin Core”)

~Fully synchronize your wallet before you begin.

This process may take some time. Bitcoin is largest, but also the best chance of your data surviving over time. If you are looking for quicker results, try the other coins.

~After the coin(s) have fully synchronized, the wallet needs the RPC (remote procedure calls) enabled. A special file called a configuration file, aka, "conf" file needs to be created and placed in a certain spot.


II. Conf File Instructions:

1. Shut down your synced coin wallet.

2. Open Notepad (located in programs/accessories).

3. Copy the following code:
server=1
rpcuser=
rpcpassword=

4. Paste into your Notepad file.

5. After rpcuser= put any user name you want (spaces are not allowed).

6. After prcpassword= put any password you want (spaces are not allowed).

7. The configuration file must be placed into a special folder associated with your wallet. (go to 8 if you haven’t done this before)
Example: C:\Users\[your user name]\AppData\Roaming\Bitcoin\bitcoin.conf
Example: C:\Users\[your user name]\AppData\Roaming\Litecoin\litecoin.conf

8. Go to File, click "Save As”.

9. At the bottom, change "Save as type" to "All Files".

10. Enter bitcoin.conf under file name. (or corresponding coin name)

11. Click on "Local Disk" (usually on the far left, towards the bottom), then "users", then , then "AppData", then "Roaming."

~You may need to enable the viewing of hidden folders and files to see the AppData Folder.
(this process is different for each version of windows, google this process if need be)
(windows 7 example: Start button - Control Panel - Appearance and Personalization - Show hidden files and folders - Remove the checkmark from the checkbox labeled “Hide extensions for known file types.” Remove the checkmark from the checkbox labeled “Hide protected operating system files.”)

12. Click save. (keep the file open on your desktop for now)


III. Wallet Configuration Instructions

1. Restart your coin wallet, wait for it to re-sync (will be quicker this time)

2. Open Apertus.

3. When open, click "Settings", then click "wallet".

4. Select the wallet you want from the top pulldown menu "Select Wallet From List".

5. At the bottom, change the "RPC User" info to the user name you created in the conf file.

6. Change the "RPC Password" info to the password you created.

7. At the bottom, make sure "Enable Wallet" is check-marked.

8. Click save in the upper right corner.

9. Restart Apertus (exit all Apertus files, exit Apertus, re-start Apertus)

Apertus is now ready to read and write data on the coins blockchain.


IV. Monitor and Search the Blockchain(s)

Prior to archiving, you may want to monitor and or search the blockchain(s) for data. This will help you see that it is working.

1. Ensure you have fully synchronized your coin, and you have rebooted it since making the conf file.

2. Ensure you have rebooted Apertus after you set the configurations for the coin.

3. Click the "Explore" tab in Apertus.

4. Copy and paste the transaction id of a known archive into the box labled “ENTER TRANSACTION ID”, Hit Enter. If found the archive will be displayed in the explorer window.

5. To begin monitoring check the "Monitor Blockchains" check box
NOTE: (the checkmark may not show up right away, click once, wait, possibly a few seconds, don’t click multiple times, it may take a moment or two.)

6. Apertus is now monitoring the blockchain(s) for data.


V. Archiving

1. Apertus uses small amounts of coin to immutably embed data onto blockchains; you will need to put some into your wallet for this to work. (we get none of this btw, it is “burned” onto the blockchain, and will be lost to time. In the event of a mistake, we recommended you don’t use your primary wallet as a safety precaution)

2. Ensure your have fully synchronized your coin and rebooted it since making the conf file.

3. Ensure you have have restarted Apertus after you set the configurations for the coin.

4. At the bottom of the archive screen, click "Select Wallet"

5. Select the wallet you are using.

6. Click "Select Account".

7. Select the account you want to use. If you haven't named your accounts in your wallet, it may appear as blank, click the blank area.

8. Go to the large main area where it says "ENTER MESSAGE HERE" - click on it.

9. Enter your message there, and/or add a file using the "attach" feature.

10. Click "SAVE" on the far right. (Have patience for files larger than 3K)

You’ve just put immutable data on a Blockchain


VI. Signing your archives

Signing an archive proves the archivist owns the signing address and that the data returned to the client matches the data archived by the signee. It does this by creating a SHA256 HASH of the entire archive. The HASH is passed in as a message to the wallets signing function to create a signature. The signature and signing address is then attached to the archive for verification by other clients. Signing your archives allows other Apertus users to trust , follow and find your archives.

1. Under the “Settings” tab, click “Wallets.”

2. In the new window, select the wallet you are working with from the pull down menu.

3. Copy and paste the address you would like to use for signing in the box labeled “Signing Address”
NOTE: The address must exist within the connected wallet.

4. Check the “Enable Signing” checkbox.

5. Click Save.


VII. Trust Center

The trust center allows users to trust or block content archived onto blockchains by establishing trusted or blocked address lists. To add an address to either list, copy and paste the address into the input field located at the top of either list box and click the + button. Click the – button to remove the address from the list. Trust options can be adjusted by checking the various option boxes located below each address list.

Trust Center Options:

~Accept all content from Trusted Address List.
Any archives signed by a trusted address will bypass Apertus’s default security features ex.(html formatting and scripting, executable files)

~Block all content from Blocked Address List.
Any archives signed by a blocked address will not be written to disk. A transaction ID not found message will be displayed if attempting to explore a blocked address.

~Block all untrusted content.
Any archive not signed by an address listed in the Trusted address list will not be written to disk. A transaction ID not found message will be displayed if attempting to explore a untrusted address.

~Block all unsigned content.
Archives that have not been signed will not be written to disk. A transaction ID not found message will be displayed if attempting to explore a unsigned transaction.


How to Increase Transaction Fee, aka Parking Fee

The immutability (unprunability) of Apertus comes from the fact that it deposits small amounts of unrecoverable coin per 20 bytes of data.
This makes a blockchain permanently bigger and puts a burden on miners and pool operators.
Apertus has adjustable settings that allow you to increase miner fees if you want to assess yourself a fee to offset this burden.

1. Under the “Settings” tab, click “Wallets.”

2. In the new box, select the wallet you are working with from the pull down menu.

3. Under Transaction Fee change the setting to any larger amount you want.
Ex. if it is set at 0.0001, you can increase it by a factor of 10 by setting it to 0.001

4. Click Save.


How to upgrade Apertus:

1. Download the newest version.

2. Copy the entire contents of the folder (control-a, then right-click, click "copy")

3. Paste the contents into the old Apertus folder (may require a search to find the original folder)

4. When prompted, click "copy and replace.”


Adding a Coin

~ Wallet must be synced prior to installing on Apertus.

~ On the main Apertus page go to 'Settings' then click on 'Wallet Settings.

~ Click the down arrow on 'Select Wallet From List.'

~ If you don't see your coins name, click any of the coins to begin customizing a new one.

~ Go to 'Wallet Name' and enter the name of the coin you are installing.

~ Click the tab 'Copy'. The name will show up in the list.

~ Enter the relavent data on this list:

- Version Byte
~ Figure out what the leading character is in the coin's address.
~ Find it on the chart at https://en.bitcoin.it/wiki/List_of_address_prefixes.
~ Look to the left at the 'Decimal version.' This is your Version Byte.
~ If it gives you an address that you don't even use and tells you it's invalid, that means you have the Version Byte wrong. (there are a few options possible, try another one on the list)

- Payload Size
~ This is done by digging in the code for the numbers OR experimentation. Enter a number close to what is defauted on Bitoin. Apertus will help, read the error & change as needed.

- Transaction Size
~ this is done by digging in the code for the numbers OR experimentation. Enter a number close to what is defauted on Bitoin. Apertus will help, read the error & change as needed.

- Transaction fee
~ this is done by digging in the code for the numbers OR experimentation. Enter a number close to what is defauted on Bitoin. Apertus will help, read the error & change as needed.

- Min Transaction
~ this is done by digging in the code for the numbers OR experimentation. Enter a number close to what is defauted on Bitoin. Apertus will help, read the error & change as needed.
~ If it says "transaction amount too small", increase Min Transaction.

- RPC Port
~ Dig in the code for it, or:
~ Exit your coin.
~ Goto the command prompt and type netstat /a
~ Write down or copy/paste all the listening ports you see. (mark, copy (enter), paste into notepad).
~ Restart your coin then enter netstat /a into the command prompt again.
~ Look for the a new 0.0.0.0:XXXXX If there are two of them, try both. One of them is it.
~ If it says "Check Wallet Settings. Value cannot be null." That usually means your RPC Port is wrong (each coin can open 2 RPC IP ports, but only one of them make Apertus go, try the other one) OR you didn't do your Conf File properly.

- RCP IP
~ 127.0.0.1 for every coin so far.

- RPC User RPC Password
~ see Conf File Instructions.

- Signing Address
~ Depends on your usage needs; see Signing your Archives.

- Troubleshooting Adding a Coin:
~ If it says "Check Wallet Settings. Value cannot be null." That usually means your RPC Port is wrong (each coin can open 2 RPC IP ports, but only one of them make Apertus go, try the other one) OR you didn't do your Conf File properly.
~ If it gives you an address that you don't even use and tells you it's invalid, that means you have the Version Byte wrong. (there are a few options possible, try another one on the list)
~ If it says "transaction amount too small", increase Min Transaction.


Notes on Safety:

~ Apertus uses small amounts of coin to immutably embed data onto blockchains; you will need to put some into your wallet for this to work. We recieve none of this, it is etched onto the blockchain as unspendable transaction outputs in patterned data. In the event of a mistake, we recommended you don’t use your primary wallet as a safety precaution, and with a low cost of coins to work with

~ By default, Apertus attempts to Filter out all Unsafe Content. Treat any posting you see with the same caution you would use with unknown emails. When monitoring blockchains, do not download or run executable files from sources you have not verified. We haven’t seen any malicious content yet, it is inevitable, but perhaps humans can change.
